GroupMatch MVP - Progress Tracker
==================================

[x] Project Setup
    - Next.js 14 with App Router
    - TypeScript configuration
    - Tailwind CSS setup
    - Firebase SDK installed
    - Environment variables template created

[x] Feature 1: Authentication
    - Firebase Auth configuration (src/lib/firebase.ts)
    - Auth context with signup/signin/signout (src/lib/AuthContext.tsx)
    - Login page (src/app/login/page.tsx)
    - Signup page (src/app/signup/page.tsx)
    - Protected routes redirect to login

[x] Feature 2: Create Group
    - Group creation form (src/app/group/page.tsx)
    - Fields: name, bio, photoUrl (text input)
    - User becomes admin
    - Stores in Firestore groups collection
    - Updates user document with groupId

[x] Feature 3: Browse Groups
    - Card UI showing other groups (src/app/browse/page.tsx)
    - Like/Pass buttons
    - Filters out user's own group
    - Filters out already-liked groups
    - Stores likes in Firestore likes collection

[x] Feature 4: Match Detection
    - On like, checks for mutual like (in browse/page.tsx)
    - Creates match entry in matches collection
    - Shows match celebration modal

[x] Feature 5: Chat View
    - Matches list page (src/app/matches/page.tsx)
    - Chat page with messages (src/app/chat/[matchId]/page.tsx)
    - Send/receive messages
    - Manual refresh button (no real-time listeners)
    - Messages stored in matches/{matchId}/messages subcollection

Firestore Collections:
- users/{uid}: email, groupId
- groups/{id}: name, bio, photoUrl, adminUserId, createdAt
- likes/{id}: fromGroupId, toGroupId, createdAt
- matches/{id}: groupIds[], createdAt
- matches/{id}/messages/{id}: groupId, text, createdAt

To Run Locally:
1. Copy .env.local.example to .env.local
2. Fill in Firebase credentials
3. npm install
4. npm run dev

To Deploy to Vercel:
1. Push to GitHub
2. Import project in Vercel
3. Add environment variables in Vercel dashboard
4. Deploy
